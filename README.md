# Winnow

> AI実行つきタスク管理ツール。本質は「タスクを片付ける道具」ではなく
> **あなたの判断アテンションを配給する道具**。

登録されたタスクの木を、AIが「自動でやる／あなたに上げる／人間がやる」に確信度つきで仕分け、
自動分は実行まで回し、**あなたのキューにはエスカレーションだけが残る**。
普段のさばきがそのまま分類器の教師信号になり、境界線を日々較正していく。
新しく覚える操作はゼロ——普段の登録・並べ替え・着手・却下が、そのまま教師信号になる。

---

## できること（現状）

- **三値分類器**：`自動 / 要確認 / 要判断` を確信度・ステークス・可逆性つきで仕分け。
- **エスカレーションだけのキュー**：自動分は畳む。一行理由・スコアバッジ・確信度がグランス可能。
- **処分＝ラベル**：やる／粒度を下げる／分類し直す／もう上げるな。普段の一手が信号。
- **タスクの木 / 粒度ラダー**：親（問い・意図）とリーフ（実行可能）を区別して持つ。
- **案件・スプリント**：案件で束ね、スプリント（期間の横断カンバン）で全タスクを案件横断表示。
- **実行器**：可逆リーフは自動着火、不可逆・高ステークスは提案＋ワンタップ承認。安く取り消せて痕跡が残る。
- **監査サンプリング / 基準率補正**：自動処理の一部を確認に混ぜ、カテゴリ別カウントで境界を較正。
- **端末の劇場**：AIはtmux常駐のインタラクティブ`claude`が処理。GUIからライブ閲覧＋実端末アタッチ。
- **MCP捕獲口**：別ツールで作業中のまま winnow にタスクを放り込む `winnow_capture`。

---

## 前提

- **Node.js 20+**
- **tmux**（AIをインタラクティブセッションで常駐させるため）
- **Claude Code CLI（`claude`）にログイン済み**

## クイックスタート

```sh
npm install
npm run build
npm start          # http://127.0.0.1:8787 で起動
```

サーバは loopback（`127.0.0.1`）のみにバインドします。状態はすべて `~/.winnow/` に保存されます。
