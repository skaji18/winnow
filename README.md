# Winnow

> AI実行つきタスク管理ツール。本質は「タスクを片付ける道具」ではなく
> **あなたの判断アテンションを配給する道具**。

登録されたタスクの木を、AIが「自動でやる／あなたに上げる／人間がやる」に確信度つきで仕分け、
自動分は実行まで回し、**あなたのキューにはエスカレーションだけが残る**。
普段のさばきがそのまま分類器の教師信号になり、境界線を日々較正していく。

日々の使い方（タスクを登録・さばく人向け）は [`docs/USER_GUIDE.md`](./docs/USER_GUIDE.md) を参照。
設計の詳細は [`REQUIREMENTS.md`](./REQUIREMENTS.md)（背骨）と
[`docs/DECISIONS.md`](./docs/DECISIONS.md)（実装の選択）を参照。

## できること（現状）

- **スプリント（期間の横断カンバン）**：スプリントはグローバルな時間箱。その期間の全タスクを案件横断でボード(未着手/進行中/レビュー/完了)表示。
- **案件（プロジェクト）**：案件で束ね、案件ごとに見せ方を `ボード(状態カンバン) / フロー(優先度・期日順)` で切替。
- **タスクの木（バックログ）**：親(問い/意図)とタスク(実行可能)を区別し、粒度ラダー(テーマ→イニシアチブ→エピック→ストーリー→タスク)で持つ。
- **三値分類器**：`自動 / 要確認 / 要判断` を確信度・ステークス・可逆性つきで仕分け。明示ルール→基準率補正の順で最終決定。
- **エスカレーションだけのキュー**：自動分は畳む。案件・優先度・期日・一行理由・スコアバッジ・確信度がグランス可能。
- **処分＝ラベル**：やる／粒度を下げる／分類し直す／もう上げるな。普段の一手が教師信号。
- **監査サンプリング**：自動処理のN%を「30秒確認」として見分けのつかない形でキューに混ぜる。
- **基準率補正**：カテゴリ別カウントで境界を倒す（締めるのは速く、緩めるのは慎重に）。
- **実行器**：可逆リーフは自動着火、不可逆/高ステークスは提案＋ワンタップ承認。自動は安く取り消せて痕跡が残る。
- **レビュー還流**：実行後のレビューをタスクとして登録し直す。
- **週次サマリ**：「今週：自動X / 上げY / あなたが覆したZ / カテゴリKを倒した」。
- **端末の劇場**：AIはtmux常駐のインタラクティブ`claude`が処理。GUIからライブ閲覧＋実端末アタッチ。

## 必要なもの

- **Node.js 20+**
- **tmux**（AIをインタラクティブセッションで常駐させるため）
- **Claude Code CLI（`claude`）にログイン済み**（サブスク座席で動かす前提）

```sh
claude --version   # 入っているか確認
tmux -V
```

## セットアップ

```sh
npm install
npm run build      # フロントをビルド
npm start          # http://localhost:8787 で起動
```

開発時（フロントのホットリロードあり）:

```sh
npm run dev        # server:8787 + web:5174。ブラウザは http://localhost:5174
```

状態はすべて `~/.winnow/`（`WINNOW_HOME`で変更可）に保存されます（SQLite + IPC作業ディレクトリ）。

## AI連携の仕組み（重要）

ヘッドレス（`claude -p`）は将来サブスク対象外になるリスクがあるため、
**インタラクティブな`claude`をtmuxに常駐**させて駆動します。

- `winnow` という tmux セッションに、`control`（分類・分解・昇格判定）と
  `worker-N`（実行）のウィンドウを作り、それぞれ`claude`を起動。
- バックエンドは画面を解析しません。**依頼ファイルを書く → セッションに「読んで結果を
  resファイルにWrite、最後にdoneファイルをWrite」と指示 → doneを検知してresを読む**、という
  ファイルI/Oで通信。画面は人間が眺める劇場専用。
- 都度 `/clear` でwarmだがステートレス（タスク管理ツール本体が状態を持つ）。

### 初回の許可について

ファイルI/Oは Write ツールしか使わないため、デフォルトは `claude --permission-mode acceptEdits`
で無人化できます。もしセッションが許可/信頼プロンプトで止まる場合は、GUIの「再調律・設定 →
AI連携」で起動コマンドを次のように変更してください（隔離された作業ディレクトリで動きます）:

```
claude --dangerously-skip-permissions
```

worker がソフト開発タスクで実際にコマンドを実行する場合も同様に調整します。

### tmuxが使えない/まず動かしたい場合

設定で **headless モード** をONにすると `claude -p` で動きます（tmux不要・検証は速い）。
将来の課金方針に注意。

### セッションの起動・確認

GUIの「セッション」タブで「セッションを起動 / 再確認」。実端末で張り付くには:

```sh
tmux attach -t winnow:worker-0
```

## 使い方の流れ（新しく覚える操作はゼロ）

1. アイテムを登録（普段のタスク登録と同じ）。
2. 分類器が disposition・確信度・一行理由・スコアを書く。
3. 「キュー」が火の海ではなくエスカレーションだけの短いリストになる。
4. 普段どおりさばく（やる／降ろす／分類し直す／もう上げるな）。
5. **そのさばきが全部ラベルになり**、境界が較正されていく。

> 序盤はキューが短くなりません。それは助けていないのではなく、線を学んでいる音です（Jカーブ）。

## 非目標・限界

この最適化は収束しません。明示基準に乗る「真ん中」は速くさばけますが、
新規・高ステークスの裾（霧の領域）には原理的に届きません。狙うバーは「正しさ」ではなく
「自分で全件目視するよりマシ、かつ誤仕分けからの回復が安い」こと。詳細は `REQUIREMENTS.md` §5。
