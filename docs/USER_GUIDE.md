# Winnow 利用者ガイド

> このガイドは **Winnowを日々使う人**（タスクを登録し、キューをさばく人）向けです。
> Winnowの考え方そのものは [`README.md`](../README.md) を、用語の定義は
> [`docs/GLOSSARY.md`](./GLOSSARY.md) を参照してください。
> サーバの起動・AI連携・運用設定は [`docs/OPERATOR_GUIDE.md`](./OPERATOR_GUIDE.md) と
> [`docs/CONFIG.md`](./CONFIG.md) にまとまっています。

---

## ひとことで言うと

登録したアイテムをAIが **自動でやる / 要確認 / 要判断** の3つに仕分けし、自動でできる分は裏で片付けます。
**あなたのキューに残るのは、あなたの判断が必要なものだけ。** そして、あなたが普段どおりさばく操作
（やる・降ろす・分類し直す…）が、そのままAIへの教師信号になります。新しく覚える操作はありません。

> 仕組みの全体像（なぜキューが短くなるのか、いわゆる「Jカーブ」）は
> [`README.md`](../README.md) と [`docs/GLOSSARY.md`](./GLOSSARY.md) に書いてあります。

### 正直に: キューはゼロにはなりません

Winnowは「全部を自動化する」道具ではありません。

- **キューは空になりません。** 自動に倒せるものが増えるほど短くはなりますが、最後に残るのは
  「本当にあなたにしか決められないこと」です。それが残るのが正常な状態です。
- **一番上流の問い（テーマ級の漠然とした方向づけ）は自動化されません。** 大きな「問い」は
  分解して小さくしていくもので、AIが勝手に決め切ることはありません。
- **使い始めはキューが短くなりません。** AIがあなたの判断の境界線を学んでいる最中だからです。
  さばき続けるほど自動に倒せるものが増えていきます。最初の手応えのなさは「線を学んでいる音」です。

## 画面の全体像

上部には常に **今週のサマリ**（1行）が出ます。ループが閉じている＝ちゃんと働いていることが
一目で分かります。状態は自動で更新されます（数秒ごと）。

タブは6つです:

| タブ | 役割 |
|---|---|
| **キュー** | 日々の主戦場。あなたの判断が要るものだけが並ぶ。 |
| **スプリント** | 「期間」の横断カンバン。その期間の全タスクを案件をまたいで状態表示。 |
| **案件** | プロジェクト単位の状態確認ビュー。 |
| **バックログ** | 全アイテムを親子・粒度つきのツリーで俯瞰。 |
| **セッション** | AIの作業の様子をライブで覗ける（普段は見なくてOK）。 |
| **再調律・設定** | 締め具合・監査率などのつまみと、学習した境界の確認。 |

画面の **登録口はどのタブにいても常に表示** されています。思いついたら即その場で放り込めます。

## 1. アイテムを登録する

画面上部のテキストエリアに会話・メモ・タスクを **雑に貼って** **「登録して分類」**
（または Ctrl/⌘+Enter）。これだけです。**タイトルは空でOK** ——本文の1行目から仮タイトルが
作られ、あとでAIが要約で上書きします。登録するとAIがすぐに仕分け・スコア付けをします。

事前に種類や大きさを決める必要はありません。ざっくりした「問い」でも、具体的な
「やること」でも、同じ口に放り込めば、AIが粒度を判定します。大きすぎるものは
キューに残り、あとで分解して小さくできます。

### 「詳細」で足せること（任意）

「詳細」を開くと次を指定できます。**上流（最初）に丁寧に書くほど、分解後の一つひとつの
タスクまで文脈が効いてきます。**

- **タイトル** — 明示したいとき。
- **案件** — どのプロジェクトに属させるか（既定は「なし」）。
- **優先度** — 緊急 / 高 / 中 / 低。キューの並び順に効きます。
- **期日** — 期限。超過・間近はキューで前に出ます。
- **領域** — 「一般」か「ソフト開発」。ソフト開発なら **作業ディレクトリ** も指定でき、
  AIがそのリポジトリで作業します。

### 別のツールから登録したいとき（MCP）

WinnowはMCPツール **`winnow_capture`** を1つだけ公開しています。対応するクライアント
（チャットや別のエージェントなど）から、本文をそのまま放り込むだけで登録できます。
キューの閲覧や決定の書き戻しはできず、**登録専用の口**です。
接続方法と引数は [`docs/MCP.md`](./MCP.md) を参照してください。

## 2. キューでさばく（毎日の中心）

キューの各カードには、タイトル・**一行の理由**・各種バッジ・確信度バーが並びます。
中身を全部読まなくても、ひと目で「なぜ上がってきたか」が分かるようになっています。

カードのボタンで処分します。**どのボタンを押したかが、そのままAIへの教師信号です。**

| ボタン | 意味 |
|---|---|
| **やる** | 自分で対応した／対応すると決めた。 |
| **実行する** | （実行可能なタスクの場合）AIに今すぐ実行させる。 |
| **分解する** | （まだ大きい問い/まとめの場合）小さなタスクに割る → [分解](#4-分解する)へ。 |
| **案件に昇格** | この問いをプロジェクト（入れ物）に格上げし、配下ごと紐付ける。上流が案件級で来たとき用。 |
| **粒度を下げる** | まだ判断には早い。一段下の扱いに落とす。 |
| **分類し直す**（プルダウン） | →自動 / →要確認 / →要判断 に手で倒す。境界線への明示的なナッジ。 |
| **もう上げるな** | この種類は今後上げず自動に倒す。「これはいちいち確認しなくていい」の意思表示。 |
| **当面は上げて** | この種類は当面エスカレ（要確認）に締める。「もう上げるな」の逆で、念のため上げ続けたいときに。 |
| **却下** | 不要。捨てる。 |

> **「もう上げるな」と「当面は上げて」は対称** です。前者は自動に倒す（緩め）、後者はエスカレに締める
> （締め）。背骨どおり **締めるのは速く、緩めるのは慎重に**: 締め方向は片手で打てますが、緩め方向は
> 正規路（「分類し直す → 自動」）から倒します。

### さばいた直後の「取り消す」（Undo）

直前の処分が間違いだったら、そのカードの下に出る **「取り消す（やる/却下/分類し直し/もう上げるな）」**
で直近1手を巻き戻せます。処分が「さばく前」の状態に戻り、教師信号からも外れます。
（締め方向の操作には汎用の取り消しは出ません。緩めるときは「分類し直す」から倒してください。）

> **確信度が低くて上がってきたものは「分類し直す」が効きます。** AIが自信を持てず「念のため」
> 要判断に上げたケースは、あなたが **「分類し直す → 自動」** に倒せば、それが境界線への明示的な
> ナッジになります。逆に自動に任せたくないものは「→要確認/要判断」に倒します。

### 承認待ち（不可逆・高ステークスなもの）

取り消しにくい操作や影響の大きい操作は、AIが勝手に実行しません。カードに **「承認待ち」**
と出て、**「承認して実行」** か **「取り消す」** を選べます。あなたのワンタップが要る安全弁です。

> **PR作成を任せたいとき（外部送信の解禁）**：設定の「外部送信(push/PR作成)を承認時に解禁」を ON にすると、
> 承認した実装タスクで AI が push／PR作成まで実行します（**マージ・本番デプロイ・削除はしません＝あなたが外で**）。
> 既定は OFF（勝手に外へ出さない安全側）です。

### 引き取り待ち（やって終わりでない成果物）

実行は成功しても、**人間が確認・採用すべき成果物が残る**もの（PRを作った／外部に出す成果物／高ステークス）は、
完了に畳まず **「引き取り待ち」** としてキューの前面に出ます。カードの **「受け取る(完了)」** で完了に進み、
**「取り消す」** もできます。CI の確認やマージは winnow の外であなたが行います（winnow は CI を見ません・マージしません）。
一方、ローカルに書くだけのような **「やって終わり」** の成果物は、これまでどおり静かに完了して畳まれます。
ヘッダの **「引き取り待ち K」** と週次の一行で、放置されていないか気づけます。

### 止まった項目の「再浮上」

実行に失敗した項目や、人手で保留中の項目は、キューに必ず（畳まずに） **「再浮上」** バッジ付きで出てきます。
カードには **「再実行 / エスカレ / 却下」** の3つが出ます。

- **再実行** — もう一度AIに実行させる。
- **エスカレ** — この種類は当面エスカレに締める（「当面は上げて」と同じ）。
- **却下** — もういらない。捨てる。

未着手のまま滞留した要確認/要判断の項目には、しばらく経つと一行理由の末尾に「◯日滞留」と付きます。
気づけるように出すだけで、機械的に止めることはしません。

### 自動で実行済みのもの

可逆（安く取り消せる）なタスクはAIが自動で実行し、結果だけがカードに出ます。
**「実行結果 / メモを見る」**（実行前の項目では **「計画プレビュー」**）で中身を確認でき、
納得いかなければ **「取り消す」** できます。
※取り消しの痕跡は履歴に残りますが、副作用そのものは自動では巻き戻りません。取り消し時には、
AIが申告した **巻き戻し手順**（変更ファイルや戻す手順の例）があればカードに表示されます
——これは案内で、自動実行はしません。必要なら手元で実行してください。

> **一般領域（ソフト開発以外）の成果物には「出口」が付きます。** 自動で仕上がった成果物は
> **「コピー」** でそのまま持ち出せ、一行の指示を添えて **「この方向で直す」** を押すと、
> 前回の成果物を踏まえてAIが直し直します。

### 「確認(自動処理)」チップが付いたカード

ときどき、見た目はふつうのカードに小さく **「確認(自動処理)」** チップが付きます。
これは **監査サンプル** ——本来は自動で畳めたものを、抜き取りであなたに見せています。
普通にさばいてください。あなたの「やる/却下…」が、自動仕分けが正しかったかの答え合わせになります。
（混ぜる割合は設定の「監査サンプル率」で変えられます。）

### 検索・絞り込み（`/` でトグル）

キューが長いときは、キーボードの **`/`** で検索・絞り込みバーを開けます（`Esc` で閉じる）。
タイトル・本文・理由へのインクリメンタル検索に加え、処分（自動/要確認/要判断）・優先度・期日
（期日超過/期日間近）・区分・案件のチップで絞り込めます。**表示を絞るだけで、並び順は変わりません。**

### キュー上部の帯

使い始め（さばいた実績がまだ少ないうち）は **「今は線を学習中です。あと N 件ほどさばくと、
自動に倒し始めます」** という帯が出ます。手応えのなさは「線を学んでいる音」です（前述）。

また「自動で畳んだ N 件はここに出していません」という帯が出ることがあります。
Winnowが裏でこれだけ片付けた、というカウントです。

### ヘッダの集計と「自動実行の一時停止」

ヘッダには **「実行中 N / 承認待ち M / 引き取り待ち K」** が出ます。実行中が並列数の上限を超えると薄く色が付きます
（止めはせず、気づかせるだけ）。「引き取り待ち K」は、実行は終わったがあなたの受領・採用が要る成果物の件数です。

その隣の **「⏸ 自動実行を一時停止」** で、AIの自動実行だけを一時的に止められます
（再開は **「▶ 自動実行を再開」**）。一時停止中でも、承認待ちの **ワンタップ承認** や手動の
**「実行する」** は通せます——止まるのは自動着火だけです。

## 3. バッジの読み方

カードやツリーに出る記号の意味です（表示ラベルと内部の用語の対応は
[`docs/GLOSSARY.md`](./GLOSSARY.md) にまとめています）:

- **仕分け** — `自動` / `要確認` / `要判断`。AIがどう扱おうとしているか。
- **確信度バー** — その仕分けへの自信（%）。低いほど「念のため上げた」寄り。
- **粒度** — テーマ → イニシアチブ → エピック → ストーリー → タスク（大きい順）。
  `親・◯◯` は束ねる側（問い/まとめ）、ラベルだけのものは実行可能なタスク。
- **優先度** — 緊急 / 高 / 中 / 低。
- **期日** — 残り日数。超過・間近は強調。
- **ステークス / 可逆性 / 区分** — 影響の大きさ・取り消しやすさ・カテゴリ。
  なぜそう仕分けたかの根拠。
- **状態** — 受信 / 未着手 / 進行中 / レビュー / 完了 / 却下 / 停滞。
- **🔗 ソース / 📎 成果物** — 登録時に原典URL（課題やPR）が付いていれば「ソース」リンクが、
  AIの成果物に外部ファイル/リンクがあれば「成果物」リンクが出ます。開くだけの参照で、
  Winnowが外部に何かを送ることはありません。

## 4. 分解する

大きな問いやまとめは、そのままでは実行できません。カードの **「分解する」** を押すと、
AIが **複数の「割り方」の選択肢** を提案します。それぞれに、

- 割り方のラベルと **理由**、
- **反復** か **一括** か（不確実な段は短いサイクルで情報を買う割り方になりがち）、
- 生成される子タスクの一覧（各子の受け入れ条件まで）

が出ます。気に入った案の **「この割り方で作る」** を押すと、子タスクが作られます。
分解のたびに詳細が積み上がり、一番下のタスクには具体的な完了条件が付きます。
情報が足りないタスクは自動に倒さず手元に上げ直されます。

### リポジトリが複数に分かれているとき（polyrepo）

プロダクトの repo が複数に分かれている場合は、**案件の「前提・文脈」に repo 一覧を書いておきます**
（例: `repos: web=/abs/path/web, api=/abs/path/api`）。分解時、AIが**別 repo で作業する子だけ**に
作業ディレクトリを割り当て、子の一覧に 📁 バッジで表示されます（合っているか目視で確認してください。
モノレポなら割り当てず、親の1つの作業ディレクトリを全子が継承します）。

> **横断的な1変更には注意。** 「APIの戻り値を変えてフロントでも使う」のように **1つの変更が
> 複数 repo にまたがる** ものは、AIに丸ごと自動実行させると契約（型・項目名）や順序がズレやすい。
> winnow は安全のため、**同じ案件で別 repo の自動実行が並んだら自動着火を止めて承認待ち**にします。
> 中身が独立した別件ならそのままワンタップで実行、横断変更なら先に契約を決めてから下流の repo に
> 着手するのが安全です。

## 5. 案件（プロジェクト）

関連するアイテムをプロジェクトで束ねられます。「案件」タブの左で案件を選ぶ／新規作成します。
**新規作成時に「最初の問い」も同時に登録** でき、作成時に **「ボード」/「フロー」** の見せ方も選べます。

各案件は **見せ方を2つ** から選べます（後から切り替え可能）:

- **ボード** — 状態カンバン（未着手→進行中→レビュー→完了…）。**カードをドラッグで列移動**。
- **フロー** — 優先度・期日順の連続リスト。継続的に流れる案件向け。

「案件の前提・文脈」を書いておくと、その案件のタスクの分解・実行にAIが反映します。

> 案件を削除しても **タスクは消えません**。案件への紐付けが外れるだけです。

---

## 6. スプリント（期間）

スプリントは案件に属さない **グローバルな時間箱（期間）** です。「スプリント」タブで期間を選ぶと、
**その期間の全タスクを案件横断で** ボード表示します（カードには所属案件のチップが付きます）。

- **カードをドラッグ** して状態（未着手/進行中/レビュー/完了）を移動。
- 下の「このスプリントに引き込む」から、未割当タスクを **案件をまたいで** この期間に取り込む。
- カードの「スプリント移動」で別期間へ移したり外したり。

> スプリントを削除しても **タスクは残り**、割当が外れるだけです。

---

## 7. バックログ

全アイテムを **親子・粒度つきのツリー** で俯瞰します。上の「案件」フィルタで絞り込めます。
`◆` は束ねる側（問い/まとめ）、`▸` は実行可能なタスク。各行で粒度・優先度・期日・仕分け・状態が見え、
親の問いをここからも「案件に昇格」したり、不要なものを「削除」できます。

## 8. 再調律・設定（つまみで線を動かす）

ここは「最適点に落ち着く画面」ではなく、**動く標的を追いながら線を調整する計器盤** です。
利用者として日々触るのは、上の2つのスライダーと前提テキストです。

- **プロダクトの前提** — 何を作っているか・方針・規約など。ここに書いた前提が
  仕分け・分解・実行すべてに効きます（最も投資対効果が高い場所）。
- **締め具合（escalation tightness）** — 高いほどエスカレ寄り（安全側）。
  **締めるのは速く、緩めるのは慎重に** が原則。誤って自動に倒したくないときは上げる。
- **監査サンプル率** — 自動処理の何%を「ちょっと確認」としてキューに混ぜるか。
  節約したい注意を、わざと少額だけ払って答え合わせをする仕組み。
- **学習した境界（明示ルール）** — さばきが溜まると現れる、自動化された境界。
  納得いかないものは「解除」できます。

> このページには **AI並列数・control/worker の起動コマンド・headless 切り替え** などの欄もありますが、
> これらは運用側の設定です。詳しくは [`docs/OPERATOR_GUIDE.md`](./OPERATOR_GUIDE.md) と
> [`docs/CONFIG.md`](./CONFIG.md) を参照してください。

## 一日の回し方（おすすめ）

1. 思いついたことは、その場の登録口にどんどん放り込む。
2. 「キュー」を開く。火の海ではなく、**あなたの判断が要る短いリスト** が並ぶ。
3. 上から順にさばく:**やる / 実行する / 分解する / 粒度を下げる / 分類し直す / もう上げるな / 当面は上げて / 却下**。
   迷ったら、その仕分けが妥当かだけ判断すればよい（中身を全部読まなくていい）。間違えたら直近1手は
   「取り消す」で巻き戻せます。
4. 「承認待ち」が出たら、内容を見て承認 or 取り消し。
5. ときどき来る「確認(自動処理)」チップ付きカードも、普段どおりさばく＝答え合わせ。
6. 週次サマリで「今週どれだけ自動化が効いたか／自分が何を覆したか」を確認。

これを続けるだけで、あなたが何も意識しなくても境界線が較正され、キューは
だんだん「本当にあなたにしか決められないこと」だけになっていきます。

## 困ったら

よくあるつまずきと対処です（より詳しくは [`docs/TROUBLESHOOTING.md`](./TROUBLESHOOTING.md) を参照）:

- **キューが全然減らない** → 序盤は正常です。さばき続けてください（前述「キューはゼロにはなりません」）。
- **これは確認したくない、というものが何度も上がる** → そのカードで「もう上げるな」、
  または設定の締め具合を少し下げる。
- **逆に、自動に任せたくないものが勝手に処理される** → 「分類し直す→要確認/要判断」で倒し、
  設定の締め具合を上げる。
- **確信度が低くて要判断に上がってきた** → 妥当なら「分類し直す→自動」に倒せば境界線に教えられます。
- **大きすぎて手がつかないアイテム** → 「分解する」で実行可能なタスクまで割る。
- **AIの実行結果が不安** → カードの「実行結果 / メモを見る」で中身を確認、必要なら「取り消す」。
- **画面上部に赤い「AI 未接続」バナーが出る** → その場の **「セッション起動」** ボタンを押すと
  AIセッションを立ち上げ直します。直らないときは [`docs/TROUBLESHOOTING.md`](./TROUBLESHOOTING.md) を参照。
- **実行に失敗した／止まった項目を立て直したい** → キュー上部に「再浮上」で上がってきます。
  「再実行 / エスカレ / 却下」から選んでください。
